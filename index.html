A Visual Introduction To Machine Learning

Some Useful Information

If you're not an expert in machine learning, it's normal to feel overwhelmed by the sheer number of algorithms that can be implemented to solve any given problem. The reason so many algorithms exist is simple: each one has certain pros and cons that allow it to work well for some tasks and poorly for others.

We're going to look at four particularly common algorithms in a way that visually expresses their core properties. To do this, we'll be using a dataset known as the Iris dataset. It is a simple table of observations for 3 particular types of flowers.


Do you think these flowers look similar or different? If I show you a new picture could you tell me which of these three flowers my new picture belongs to? In machine learning this challenge is known as classification. The Iris dataset challenges you further. Instead of looking at pictures of flowers, you're only given the lengths and widths of the sepals and petals of these flowers. Here are three rows directly from the Iris dataset:
Record ID Sepal Length (cm) Sepal Width (cm) Petal Length (cm) Petal Width (cm) Flower Type
1 5.1 3.5 1.4 0.2 Iris vetosa
2 7.0 3.2 4.7 1.4 Iris versicolor
3 6.3 3.3 6.0 2.5 Iris virginica
Now can you tell what flower this is?
Record ID Sepal Length (cm) Sepal Width (cm) Petal Length (cm) Petal Width (cm) Flower Type
4 5.7 3.8 1.7 0.3 ??????????
It's Iris setosa! Whether you got it right or not, imagine you had a thousand or even a million of these records that you needed to label this same way. The whole Iris dataset is only 150 rows, but even at that number we'd be cringing. Luckily, as you may have guessed, this task is perfect for machine learning algorithms!

So back to the plethora of algorithms that might help us with our flower classification task, we can make things easier by dividing all of them up into two categories:
  • Supervised learning algorithms - (e.g. k-nearest neighbors, decision trees)
    • These algorithms make use of prior data to label new data. So we give them some flower records with known labels and based on that, they decide the proper label for new unlabeled flower records.
  • Unsupervised learning algorithms - (e.g. k-means)
    • The big difference for these algorithms is that they don't rely on known examples to label new ones. Imagine I gave you all 150 rows of the Iris dataset without the "flower type" label and asked, "how many flower types are there?". You'd obviously say three because you're familiar with the dataset, but if it were some new kind of dataset, there could be anywhere from 1-150 types of flowers, right?
